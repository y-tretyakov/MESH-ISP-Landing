## Обзор проекта

MESH - веб-сайт интернет-провайдера, предлагающий высокоскоростное подключение к интернету в городе Чугуев (Харьковская область). Сайт представляет информацию о тарифах, компании, включает публичный договор и интегрирует AI-помощника на базе Google Gemini для консультации пользователей.

### Технологии

- **Frontend**: HTML, CSS, JavaScript
- **Фреймворки**: TailwindCSS (через CDN)
- **Интеграции**: Google Gemini API
- **Визуальные эффекты**: Canvas для анимации частиц
- **Видео**: HTML5 Video для фоновых видео

## Структура проекта


   ## Руководство по развертыванию

   ### Подготовка к деплою

   1. **Клонирование репозитория**:
   ```bash
   git clone <url-репозитория>
   cd MESH
   ```

   2. **Замена API ключей**:
   - Откройте `index.html`
   - Найдите строку с `const apiKey`
   - Замените значение на ваш собственный ключ от Google AI Studio

   3. **Проверка всех ссылок и контактной информации**:
   - Убедитесь, что все контактные данные актуальны
   - Проверьте ссылки на внешние ресурсы (например, на личный кабинет)

   ### Вариант 1: Деплой на обычный хостинг

   1. **Загрузка файлов на хостинг**:
   - Используйте FTP-клиент для загрузки всех файлов проекта
   - Убедитесь, что файл `index.html` находится в корневой директории
   - Проверьте, что видеофайл `mesh-main-baner.mp4` успешно загружен

   2. **Настройка MIME-типов** (если необходимо):
   - Убедитесь, что ваш хостинг правильно обрабатывает .mp4 файлы
   - При необходимости, создайте или отредактируйте .htaccess файл:
     ```
     AddType video/mp4 .mp4
     AddType video/webm .webm
     ```

   3. **Настройка SSL** (рекомендуется):
   - Активируйте SSL-сертификат через панель управления хостингом
   - Настройте перенаправление с HTTP на HTTPS

   ### Вариант 2: Деплой на GitHub Pages

   1. **Создание репозитория на GitHub**:
   - Создайте новый репозиторий на GitHub
   - Загрузите все файлы проекта

   2. **Настройка GitHub Pages**:
   - Перейдите в настройки репозитория
   - В разделе GitHub Pages выберите ветку (обычно `main`)
   - Сохраните настройки и дождитесь завершения деплоя

   3. **Настройка домена** (опционально):
   - В разделе GitHub Pages добавьте собственный домен
   - Настройте DNS-записи у вашего регистратора доменов

   ### Вариант 3: Деплой на Netlify

   1. **Создание аккаунта на Netlify**:
   - Зарегистрируйтесь на [Netlify](https://www.netlify.com/)
   - Подключите ваш GitHub аккаунт

   2. **Деплой сайта**:
   - Выберите "New site from Git"
   - Выберите репозиторий с проектом
   - Настройки сборки оставьте по умолчанию
   - Нажмите "Deploy site"

   3. **Настройка переменных окружения** (для API ключа):
   - В настройках сайта перейдите в раздел "Environment variables"
   - Добавьте переменную `GEMINI_API_KEY` с вашим ключом
   - Обновите код в `index.html` для использования переменной:
     ```javascript
     const apiKey = process.env.GEMINI_API_KEY || "ваш_резервный_ключ";
     ```

     ## Оптимизация сайта

     ### Оптимизация изображений и видео

     1. **Видео**:
        - Оптимизируйте видеофайл `mesh-main-baner.mp4` с помощью инструментов типа HandBrake
        - Рассмотрите возможность добавления WebM версии для лучшей совместимости

     2. **Изображения**:
        - Оптимизируйте все изображения с помощью инструментов типа TinyPNG
        - Используйте современные форматы WebP с резервными копиями

     ### Оптимизация производительности

     1. **Кэширование**:
        - Добавьте заголовки кэширования для статических ресурсов
        - Создайте файл `.htaccess` с правилами кэширования:
     ```
     <IfModule mod_expires.c>
       ExpiresActive On
       ExpiresByType text/css "access plus 1 year"
       ExpiresByType application/javascript "access plus 1 year"
       ExpiresByType video/mp4 "access plus 1 month"
     </IfModule>
     ```

     2. **Минимизация JavaScript**:
        - Рассмотрите возможность минимизации встроенного JavaScript
        - Внедрите асинхронную загрузку для неблокирующих скриптов

     ## Устранение неполадок

     ### Проблемы с AI-интеграцией

     1. **Ошибка "API Key not valid"**:
        - Проверьте правильность API ключа
        - Убедитесь, что ключ активирован в Google AI Studio
        - Проверьте ограничения по использованию ключа (квоты)

     2. **Медленные ответы от AI**:
        - Добавьте индикаторы загрузки во время ожидания ответа
        - Рассмотрите возможность кэширования частых вопросов

     ### Проблемы с видео

     1. **Видео не воспроизводится**:
        - Проверьте правильность пути к файлу
        - Убедитесь, что формат поддерживается браузером
        - Добавьте резервный формат (WebM) или статичное изображение

     2. **Высокая нагрузка на мобильных устройствах**:
        - Добавьте определение типа устройства и загрузку оптимизированной версии для мобильных
        - Рассмотрите возможность использования статичного изображения вместо видео на мобильных
